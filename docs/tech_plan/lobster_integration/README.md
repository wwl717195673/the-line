# 龙虾集成技术方案

基于《虾线龙虾集成产品方案》(docs/prd/龙虾集成产品方案.md)，本目录包含龙虾集成的完整技术设计文档。

## 方案概览

采用**分层推进**策略，将 MVP 拆为 4 个可独立交付的阶段：

| 阶段 | 文档 | 内容 | 依赖 |
|------|------|------|------|
| P1 | [01_data_layer.md](./01_data_layer.md) | 数据模型扩展 | 无 |
| P2 | [02_agent_execution.md](./02_agent_execution.md) | 龙虾真实执行 | P1 |
| P3 | [03_draft_orchestration.md](./03_draft_orchestration.md) | 龙虾辅助编排 | P1 |
| P4 | [04_frontend_integration.md](./04_frontend_integration.md) | 前端集成 | P2 + P3 |

## 阶段依赖图

```
P1 (数据层)
  ├── P2 (真实执行) ──┐
  └── P3 (辅助编排) ──┼── P4 (前端集成)
```

P2 和 P3 可并行推进。P4 依赖 P2 + P3 的接口完成。

## 设计原则

1. **龙虾负责执行，人负责结果** — 产品核心理念贯穿数据模型和交互设计
2. **协议先行，Mock 验证** — 龙虾侧先设计接口协议 + Mock 实现，后续替换真实服务
3. **向后兼容** — 保留现有节点类型和固定模板，新能力增量扩展
4. **执行主体与结果责任人分离** — 每个节点显式区分谁执行、谁负责
5. **流程状态只能由虾线推进** — 龙虾只能提交回执，不能直接改流程状态
6. **自动节点单飞执行** — 同一节点同一时刻只允许一个活跃 AgentTask，避免重复副作用
