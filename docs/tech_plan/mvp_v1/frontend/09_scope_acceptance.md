# 09 前端范围与验收

---

## 1. 模块目标

明确 MVP V1 前端做什么、不做什么，以及前端总体验收标准。

---

## 2. 前端必做范围

页面：

* 工作台
* 模板列表页
* 模板详情页
* 流程发起页
* 流程实例列表页
* 流程详情页
* 节点详情页或节点详情侧栏
* 人员管理页
* 龙虾管理页
* 交付中心列表页
* 交付结果页

能力：

* 查询和展示固定模板
* 发起流程实例
* 展示流程实例列表
* 展示流程详情和节点时间线
* 展示和处理节点详情
* 评论发布和展示
* 附件上传和展示
* 节点日志展示
* 龙虾模拟执行触发和结果展示
* 交付物生成和验收
* 人员和龙虾基础管理

---

## 3. 前端不做范围

不做页面：

* 拖拽式模板设计器
* BPMN 画布
* 高级 BI 报表
* 完整通知中心
* 审计日志后台
* 移动端专项页面
* PDF / Excel 导出页面

不做交互：

* 节点拖拽排序
* 节点连线编辑
* 条件分支可视化配置
* 并行节点编排
* 评论编辑和删除
* 附件在线预览和在线编辑
* 真实消息通知
* 真实 OpenClaw 执行进度流式展示

---

## 4. 前端开发顺序

### 第一阶段：壳和基础数据

1. 搭建前端工程
2. 实现应用布局和路由
3. 实现 API client
4. 实现状态枚举和公共状态标签
5. 实现人员管理
6. 实现龙虾管理

### 第二阶段：模板和流程

1. 实现模板列表
2. 实现模板详情
3. 实现流程发起页
4. 实现流程列表
5. 实现流程详情和节点时间线

### 第三阶段：节点处理

1. 实现节点详情
2. 实现节点输入表单
3. 实现节点操作按钮
4. 实现审核、驳回、补材料、异常弹窗
5. 实现龙虾模拟执行展示

### 第四阶段：协同和交付

1. 实现评论
2. 实现附件
3. 实现日志时间线
4. 实现交付物生成
5. 实现交付中心和交付详情
6. 实现交付验收

---

## 5. 前端总体验收标准

* 用户能从工作台进入发起流程
* 用户能看到固定模板并发起流程
* 发起成功后能进入流程详情页
* 流程详情能展示 9 个节点时间线
* 当前节点高亮正确
* 用户能在节点详情中暂存、提交、审核、驳回、补材料、完成和标记异常
* 节点操作成功后页面状态刷新正确
* 评论能发布并展示在正确目标下
* 附件能上传并展示在正确目标下
* 节点日志能展示关键操作
* 执行节点能触发龙虾模拟执行并展示结果
* 最后节点完成后能生成交付物
* 交付页能展示最终结果并完成验收
* 已取消流程和已完成节点在前端只读
* 前端不展示 MVP 明确不做的功能入口
