# 虾线 MVP V1 后端技术方案

---

## 1. 文档目的

本目录基于 `docs/prd/mvp_v1/` 的模块化 PRD 编写，目标是把产品需求转成后端可执行的技术方案。

后端技术栈固定为：

* 语言：Golang
* HTTP 框架：Gin
* ORM：GORM
* 数据库：MySQL 或兼容 MySQL 的关系型数据库

## 3. 后端主链路

1. 初始化人员、龙虾和固定模板
2. 用户调用发起流程接口
3. 后端基于模板创建 `flow_run` 和 9 个 `flow_run_node`
4. 前端查询流程详情，后端返回节点时间线和当前节点
5. 用户调用节点处理接口
6. 后端校验权限、状态、输入和附件
7. 后端更新节点状态并写入节点日志
8. 节点完成后，后端按 `sort_order` 激活下一个节点
9. 最后节点完成后，后端更新流程为 `completed`
10. 用户生成交付物并执行验收

---

## 4. 全局约束

* MVP 只支持固定模板
* MVP 只支持串行流程
* MVP 不做通用 BPMN 引擎
* MVP 不调用真实 OpenClaw
* MVP 不调用真实甩班工具
* 所有节点关键动作必须写日志
* 所有写操作必须做权限和状态校验
* 涉及多表更新的流程和节点操作必须使用事务
