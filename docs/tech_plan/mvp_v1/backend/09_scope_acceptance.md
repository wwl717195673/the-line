# 09 后端范围与验收

---

## 1. 模块目标

明确 MVP V1 后端做什么、不做什么、开发顺序和总体验收标准。

---

## 2. 后端必做范围

必须实现：

* Gin API 服务
* GORM 模型和数据库迁移
* 人员和龙虾基础 CRUD
* 固定模板初始化和查询
* 基于固定模板发起流程
* 自动生成 9 个实例节点
* 流程列表、详情和取消
* 工作台最近动态接口
* 节点详情查询
* 节点输入暂存
* 节点提交确认
* 审核通过、驳回、要求补材料
* 标记完成、标记异常
* 龙虾模拟执行
* 评论、附件、节点日志
* 交付物生成、查询和验收
* 统一错误响应
* 写操作事务控制

---

## 3. 后端不做范围

明确不做：

* 通用 BPMN 引擎
* 拖拽模板编排服务
* 条件分支执行引擎
* 并行节点调度
* 真实 OpenClaw 调用
* 真实甩班工具调用
* 完整通知投递系统
* 完整审计日志查询系统
* 多工作空间深度隔离
* 多租户计费
* PDF / Excel 导出
* 移动端专用接口

---

## 4. 后端开发顺序

### 第一阶段：工程和基础模型

1. 初始化 Go module
2. 初始化 Gin 服务
3. 初始化 GORM 和 MySQL 连接
4. 建立统一响应和错误处理
5. 建立 GORM 模型和迁移
6. 实现人员和龙虾基础 CRUD

### 第二阶段：模板和流程

1. 实现固定模板初始化
2. 实现模板查询
3. 实现流程发起
4. 实现流程列表
5. 实现流程详情
6. 实现流程取消

### 第三阶段：节点状态机

1. 实现节点详情
2. 实现节点输入暂存
3. 实现节点提交确认
4. 实现审核通过
5. 实现驳回和补材料
6. 实现标记完成和串行推进
7. 实现标记异常
8. 实现龙虾模拟执行

### 第四阶段：协同和交付

1. 实现评论
2. 实现附件
3. 实现节点日志
4. 实现交付物生成
5. 实现交付列表和详情
6. 实现交付验收

---

## 5. 后端总体验收标准

* 服务能启动并连接数据库
* 迁移后具备 MVP 所需 10 张主表
* 固定模板初始化可重复执行
* 模板查询能返回“班主任甩班申请”和 9 个节点
* 发起流程能创建 1 条实例和 9 条节点
* 流程列表和详情返回前端所需字段
* 节点提交、审核、驳回、补材料、完成、异常状态流转正确
* 节点完成后能自动激活下一个节点
* 最后节点完成后流程状态变为 `completed`
* 评论能绑定流程或节点
* 附件能绑定流程、节点、评论或交付物
* 节点关键动作都有日志
* 龙虾模拟执行能写入输出和日志
* 已完成流程能生成交付物
* 交付物能验收通过或驳回
* 所有写操作都有权限和状态校验
* 涉及多表更新的操作使用事务
