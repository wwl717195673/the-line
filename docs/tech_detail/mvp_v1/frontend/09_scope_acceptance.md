# 09 前端范围与验收实现总结

## 实现范围结论

已按 `docs/tech_plan/mvp_v1/frontend/09_scope_acceptance.md` 完成 MVP V1 前端范围收口与验收对齐，当前实现覆盖“必做范围”并保持“不做范围”无入口暴露。

## 必做页面对齐

已实现页面：

- 工作台：`/`（`DashboardPage`）
- 模板列表页：`/templates`
- 模板详情页：`/templates/:templateId`
- 流程发起页：`/templates/:templateId/start`
- 流程实例列表页：`/runs`、`/runs/mine`、`/runs/todo`
- 流程详情页（含节点详情侧栏）：`/runs/:runId`
- 人员管理页：`/resources/persons`
- 龙虾管理页：`/resources/agents`
- 交付中心列表页：`/deliverables`
- 交付结果页：`/deliverables/:deliverableId`

## 必做能力对齐

已对齐能力：

- 固定模板查询与展示
- 发起流程实例并跳转流程详情
- 流程列表展示与范围筛选
- 流程详情与节点时间线展示
- 节点详情处理（暂存、提交、审核、驳回、补材料、完成、异常）
- 评论发布与展示（流程/节点隔离）
- 附件上传与展示（流程/节点/交付）
- 节点日志展示
- 龙虾模拟执行触发与结果展示（执行节点）
- 交付物生成与验收
- 人员与龙虾基础管理

## 验收项对齐

按“总体验收标准”对照：

- 工作台可进入发起链路：已支持（跳转模板中心发起）
- 固定模板可见并可发起：已支持
- 发起成功进入流程详情：已支持
- 流程详情展示节点时间线：已支持（后端返回 9 节点时可完整展示）
- 当前节点高亮：已支持
- 节点动作与刷新：已支持（动作后统一刷新 run/node/comment/attachment/log）
- 评论/附件目标隔离：已支持
- 节点日志展示关键操作：已支持
- 执行节点运行龙虾并展示结果：已支持
- 最后节点完成后生成交付物入口：已支持
- 交付页结果展示与验收：已支持
- 已取消流程和已完成节点只读：已支持
  - 本次补充：节点只读态下禁用评论“标记已解决”
- 不做功能入口不暴露：已支持

## 不做范围核对

前端未提供以下入口：

- 拖拽式模板设计器 / BPMN 画布
- BI 报表 / 审计后台 / 通知中心
- 移动端专项页面
- PDF / Excel 导出页
- 节点拖拽排序、连线编辑、并行编排等高级编排能力
- 评论编辑/删除、附件在线预览/编辑、真实消息通知、流式执行进度
