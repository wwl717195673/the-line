# 虾线 MVP V1 模块化 PRD

本目录由 `docs/prd/mvp_v1.md` 拆分而来。拆分目标是让每个模块都能直接支持后续设计、前端开发、后端开发和测试验收。

---

## 1. 拆分原则

每个模块文档都包含以下内容：

* 模块目标：说明这个模块解决什么问题
* 用户与入口：说明谁会使用、从哪里进入
* 前端页面：说明页面结构、字段、操作和交互规则
* 后端功能：说明接口、服务逻辑、状态变更和数据写入
* 数据落点：说明主要表和关键字段
* 业务规则：说明实现时必须遵守的约束
* 验收标准：说明如何判断模块完成

MVP V1 的核心约束不变：先跑通“班主任甩班申请”固定流程，不做通用流程平台。

## 3. MVP 主链路

1. 管理员维护人员和龙虾基础数据
2. 系统预置“班主任甩班申请”模板
3. 发起人基于模板发起流程实例
4. 系统生成 9 个实例节点
5. 当前节点责任人在节点详情中处理输入、附件、评论和操作
6. 审核人执行通过、驳回、要求补材料
7. 系统按 `sort_order` 串行推进到下一个节点
8. 最后一个节点完成后流程进入 `completed`
9. 用户生成交付物并完成验收

## 4. 全局状态约定

### 流程实例状态

* `draft`：草稿
* `running`：进行中
* `waiting`：等待人工处理
* `blocked`：阻塞中
* `completed`：已完成
* `cancelled`：已取消

### 节点状态

* `not_started`：未开始
* `ready`：待处理
* `running`：执行中
* `waiting_confirm`：待确认
* `waiting_material`：待补材料
* `rejected`：已驳回
* `done`：已完成
* `failed`：已失败

---

## 5. 全局不做项摘要

MVP V1 不做复杂 BPMN、拖拽式模板设计器、复杂分支、深度外部系统集成、真实甩班工具调用、真实 OpenClaw 执行、完整通知中心、完整审计日志、模板版本管理、移动端和 PDF / Excel 导出。

详细说明见 `09_scope_acceptance.md`。
