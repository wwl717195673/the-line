# 06 交付模块

---

## 1. 模块目标

交付模块负责把已完成流程沉淀为可查看、可验收、可归档的交付物。

MVP 的关键不是“流程节点都完成了”，而是最终能明确交付了什么、谁验收、是否通过。

---

## 2. 用户与入口

使用角色：

* 发起人
* 管理员
* 最终节点责任人
* 验收人
* 观察者

页面入口：

* 流程详情 - 生成交付物
* 交付中心 - 全部交付
* 交付中心 - 待验收
* 交付中心 - 已归档
* 交付结果页

---

## 3. 前端页面需求

### 3.1 生成交付物入口

展示条件：

* 流程状态为 `completed`
* 当前用户是发起人、管理员或最终节点责任人
* 当前流程尚未生成交付物，或允许查看已有交付物

交互规则：

* 未完成流程不展示生成入口
* 点击后进入交付物生成表单

### 3.2 交付物生成表单

字段：

* 交付标题，必填
* 交付摘要，必填
* 关键结论，必填
* 异常说明，选填
* 验收人，必填
* 关键附件，选填

默认值：

* 交付标题默认使用流程标题加“交付结果”
* 节点完成情况由系统自动汇总
* 关键附件可以从流程和节点附件中选择

交互规则：

* 必填字段缺失时不能生成
* 生成成功后跳转交付结果页

### 3.3 交付结果页

展示内容：

* 交付标题
* 流程摘要
* 流程发起人
* 流程完成时间
* 节点完成情况
* 关键结论
* 异常说明
* 关键附件
* 验收人
* 验收状态
* 验收意见

操作项：

* 验收通过
* 验收驳回

交互规则：

* 交付页不允许修改历史节点记录
* 验收操作只改变交付物状态，不改变流程状态
* MVP 阶段不提供 PDF / Excel 导出

### 3.4 交付中心列表页

列表字段：

* 交付标题
* 关联流程
* 发起人
* 验收人
* 验收状态
* 创建时间
* 操作

筛选项：

* 验收状态
* 验收人

## 4. 后端功能需求

### 4.1 生成交付物

接口建议：`POST /api/deliverables`

输入：

* `run_id`
* `title`
* `summary`
* `result_json`
* `reviewer_person_id`
* `attachment_ids`

处理规则：

* 校验流程存在
* 校验流程状态为 `completed`
* 校验当前用户是发起人、管理员或最终节点责任人
* 校验交付标题、摘要、关键结论和验收人
* 汇总流程节点完成情况
* 汇总关键附件
* 创建 `deliverable`
* 对 `attachment_ids` 对应的附件创建交付物附件绑定
* 初始 `review_status = pending`

关键附件绑定规则：

* 不新增独立交付附件关联表
* 不移动原附件记录，原附件仍保留在流程、节点或评论下
* 对用户选中的 `attachment_ids`，复制附件元信息并创建新的 `attachment` 记录
* 新记录使用 `target_type = deliverable`、`target_id = deliverable.id`
* 新记录复用原 `file_url`，不复制文件二进制内容

### 4.2 查询交付详情

接口建议：`GET /api/deliverables/{id}`

返回内容：

* 交付基础信息
* 关联流程信息
* 节点完成情况
* 关键附件
* 验收状态
* 验收意见

### 4.3 查询交付列表

接口建议：`GET /api/deliverables`

查询参数：

* `review_status`
* `reviewer_person_id`

处理规则：

* 默认按创建时间倒序
* 待验收列表筛选 `review_status = pending`
* 已归档列表筛选 `review_status = approved`

### 4.4 验收交付物

接口建议：`POST /api/deliverables/{id}/review`

输入：

* `review_status`
* `review_comment`

处理规则：

* 校验交付物存在
* 校验当前用户是验收人或管理员
* `review_status` 只能是 `approved` 或 `rejected`
* 写入验收人
* 写入验收时间
* 写入验收意见
* 不改变流程状态

---

## 5. 数据落点

### 5.1 交付表 `deliverable`

| 字段 | 说明 |
|---|---|
| `id` | 主键 |
| `run_id` | 流程实例 ID |
| `title` | 交付标题 |
| `summary` | 交付摘要 |
| `result_json` | 结构化结果 |
| `reviewer_person_id` | 验收人 |
| `review_status` | pending / approved / rejected |
| `reviewed_at` | 验收时间 |
| `created_at` | 创建时间 |
| `updated_at` | 更新时间 |

---

## 6. 业务规则

* 只有已完成流程才能生成交付物
* 未完成流程不能生成交付物
* 交付物生成后不修改历史节点记录
* 验收通过后状态变为 `approved`
* 验收驳回后状态变为 `rejected`
* 验收驳回不自动重开流程
* MVP 阶段交付页在线查看，不做导出

---

## 7. 验收标准

* 已完成流程能生成交付物
* 未完成流程不能生成交付物
* 交付页能展示流程摘要、关键结论、节点完成情况和关键附件
* 验收人能执行验收通过
* 验收人能执行验收驳回
* 验收操作后状态立即更新
* 验收操作不改变流程状态
* 交付中心能按待验收和已归档查看交付物
