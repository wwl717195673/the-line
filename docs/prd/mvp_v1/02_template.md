# 02 模板模块

---

## 1. 模块目标

模板模块负责承载固定流程定义。MVP V1 只支持一个预置模板：**班主任甩班申请**。

模板模块不是模板设计器。它的目标是让用户知道系统支持哪条流程，并让后端可以基于模板节点生成流程实例节点。

---

## 2. 用户与入口

使用角色：

* 管理员
* 发起人
* 观察者

页面入口：

* 模板中心 - 固定模板
* 工作台 - 发起流程
* 流程发起页 - 选择模板

---

## 3. 前端页面需求

### 3.1 模板列表页

页面目标：展示可发起的固定模板。

列表字段：

* 模板名称
* 模板编码
* 模板版本
* 模板分类
* 模板状态
* 模板说明
* 更新时间
* 操作

操作项：

* 查看详情
* 使用模板

交互规则：

* MVP 阶段只展示已发布模板
* MVP 阶段只要求展示“班主任甩班申请”
* 不展示新建模板、复制模板、编辑模板、发布模板、下线模板入口
* 点击“使用模板”进入流程发起页

### 3.2 模板详情页

页面目标：让用户在发起前了解流程结构。

展示内容：

* 模板名称
* 模板说明
* 模板版本
* 节点清单
* 节点顺序
* 节点类型
* 默认责任人规则
* 默认绑定龙虾

交互规则：

* 页面只读
* 节点按 `sort_order` 展示
* 不提供拖拽排序
* 不提供节点新增、删除、编辑
* 不提供连线配置

---

## 4. 后端功能需求

### 4.1 查询模板列表

接口建议：`GET /api/templates`

查询参数：

* `status`
* `keyword`

处理规则：

* MVP 阶段默认只返回 `published` 模板
* 返回模板基础信息
* 不返回草稿和下线模板

返回字段：

* `id`
* `name`
* `code`
* `version`
* `category`
* `description`
* `status`
* `updated_at`

### 4.2 查询模板详情

接口建议：`GET /api/templates/{id}`

处理规则：

* 查询 `flow_template`
* 查询对应 `flow_template_node`
* 节点按 `sort_order` 升序返回
* 返回默认责任人规则和默认绑定龙虾

返回字段：

* 模板基础信息
* 节点清单
* 节点输入结构
* 节点输出结构
* 节点配置

### 4.3 预置模板初始化

执行方式：初始化脚本或后台管理命令。

处理规则：

* 创建 `flow_template` 记录
* 创建 9 条 `flow_template_node` 记录
* 模板状态设置为 `published`
* 模板编码固定，如 `teacher_class_transfer`
* 每个节点必须有唯一 `node_code`
* 每个节点必须有稳定 `sort_order`
* 节点类型只使用 `manual`、`review`、`notify`、`execute`、`archive`

### 4.4 模板变更限制

处理规则：

* MVP 阶段不提供前台模板编辑接口
* 如需调整模板，只能通过后台脚本处理
* 模板调整只影响新发起的流程实例
* 历史实例节点不随模板变化自动改变
* `flow_run.template_version` 需要记录发起时使用的模板版本

---

## 5. 后端生成实例节点规则

当用户基于模板发起流程时，后端需要执行以下逻辑：

* 校验模板存在
* 校验模板状态为 `published`
* 查询模板节点并按 `sort_order` 排序
* 创建 `flow_run`
* 基于每个模板节点创建 `flow_run_node`
* 第一个实例节点状态为 `ready`
* 其他实例节点状态为 `not_started`
* 实例节点复制模板节点的名称、编码、类型、默认责任人、默认龙虾
* 流程实例 `current_node_code` 设置为第一个节点编码
* 流程实例 `current_status` 设置为 `running`

---

## 6. 数据落点

### 6.1 流程模板表 `flow_template`

| 字段 | 说明 |
|---|---|
| `id` | 主键 |
| `name` | 模板名 |
| `code` | 模板编码 |
| `version` | 版本号 |
| `category` | 分类 |
| `description` | 模板说明 |
| `status` | draft / published / offline |
| `created_by` | 创建人 |
| `created_at` | 创建时间 |
| `updated_at` | 更新时间 |

### 6.2 模板节点表 `flow_template_node`

| 字段 | 说明 |
|---|---|
| `id` | 主键 |
| `template_id` | 模板 ID |
| `node_code` | 节点编码 |
| `node_name` | 节点名称 |
| `node_type` | 节点类型 |
| `sort_order` | 顺序 |
| `default_owner_rule` | 默认责任人规则 |
| `default_agent_id` | 默认龙虾 |
| `input_schema_json` | 输入结构 |
| `output_schema_json` | 输出结构 |
| `config_json` | 节点配置 |

---

## 7. 业务规则

* MVP 只支持固定模板
* MVP 不做模板前台编辑
* MVP 不做拖拽编排
* MVP 不做并行节点
* MVP 不做条件分支
* MVP 节点顺序完全由 `sort_order` 决定
* 模板变更不影响已经发起的历史流程
* 模板详情页必须只读

---

## 8. 验收标准

* 模板中心能看到“班主任甩班申请”
* 模板详情能看到 9 个节点
* 节点顺序与固定流程一致
* 模板详情不出现编辑、拖拽、发布、下线入口
* 发起流程时能基于模板生成 9 个实例节点
* 历史流程实例不受后续模板调整影响
