# 05 协同留痕模块

---

## 1. 模块目标

协同留痕模块负责把评论、附件和关键动作日志沉淀在流程和节点上下文里。

MVP 阶段不做完整通知中心，也不做完整审计后台。重点是让业务沟通、补充材料和处理记录不再散落在群消息中。

---

## 2. 用户与入口

使用角色：

* 发起人
* 节点责任人
* 审核人
* 协作者
* 管理员
* 观察者

页面入口：

* 流程详情 - 流程评论区
* 节点详情 - 节点评论区
* 节点详情 - 附件列表
* 节点详情 - 执行日志
* 交付页 - 关键附件

---

## 3. 前端页面需求

### 3.1 流程评论区

页面目标：承载对整条流程的全局说明和结论沉淀。

展示内容：

* 评论作者
* 评论内容
* 创建时间
* 已解决状态

操作项：

* 发表评论
* 标记已解决

交互规则：

* 评论内容不能为空
* 流程评论绑定到流程实例
* 流程评论不展示在节点评论区
* MVP 阶段不做评论编辑和删除
* MVP 阶段 `@人` 先作为普通文本保存

### 3.2 节点评论区

页面目标：承载围绕具体节点的沟通、补材料说明、驳回原因和结论确认。

展示内容：

* 评论作者
* 评论内容
* 创建时间
* 已解决状态

操作项：

* 发表评论
* 标记已解决
* 上传评论附件

交互规则：

* 节点评论绑定到具体实例节点
* 节点评论不能串到其他节点
* 协作者可以评论和上传补充材料
* 观察者只读

### 3.3 附件列表

页面目标：展示节点、评论、流程或交付物绑定的附件。

展示字段：

* 文件名
* 文件类型
* 文件大小
* 上传人
* 上传时间
* 下载或查看入口

交互规则：

* 附件必须绑定到一个目标对象
* 节点附件只显示在对应节点下
* 评论附件只显示在对应评论下
* 交付附件只显示在交付页关键附件区域
* MVP 阶段不做在线预览和在线编辑

### 3.4 节点日志区

页面目标：让用户看到节点关键动作的时间线。

展示字段：

* 日志类型
* 操作者类型
* 操作者名称
* 日志内容
* 创建时间

交互规则：

* 日志只读
* 日志不能删除
* 龙虾模拟执行日志需要和人工操作日志区分

---

## 4. 后端功能需求

### 4.1 发表评论

接口建议：`POST /api/comments`

输入：

* `target_type`
* `target_id`
* `content`

处理规则：

* 校验目标对象存在
* 校验用户有目标对象查看权限
* 校验评论内容不能为空
* 创建 `comment` 记录
* MVP 阶段不解析真实 mention
* MVP 阶段不发送通知

### 4.2 查询评论列表

接口建议：`GET /api/comments`

查询参数：

* `target_type`
* `target_id`

处理规则：

* 校验目标对象权限
* 只返回目标对象下的评论
* 按创建时间排序

### 4.3 标记评论已解决

接口建议：`POST /api/comments/{id}/resolve`

处理规则：

* 校验评论存在
* 校验用户有处理权限
* 更新 `comment.is_resolved = 1`
* 不改变流程和节点状态

### 4.4 上传附件

接口建议：`POST /api/attachments`

输入：

* `target_type`
* `target_id`
* `file`

处理规则：

* 校验目标对象存在
* 校验用户对目标对象有上传权限
* 保存文件
* 创建 `attachment` 记录
* 写入节点日志，如果附件目标是节点

### 4.5 查询附件列表

接口建议：`GET /api/attachments`

查询参数：

* `target_type`
* `target_id`

处理规则：

* 校验目标对象查看权限
* 只返回绑定到目标对象的附件
* 不返回其他节点或其他评论的附件

### 4.6 写入节点日志

内部服务建议：`NodeLogService.append`

触发动作：

* 创建流程
* 暂存节点输入
* 提交确认
* 审核通过
* 驳回
* 要求补材料
* 上传附件
* 运行龙虾
* 标记异常
* 标记完成
* 取消流程

处理规则：

* 日志只追加，不编辑
* 日志必须记录操作人
* 日志必须记录操作时间
* 日志必须记录动作内容
* 系统和龙虾动作需要使用不同 `operator_type`

## 5. 数据落点

### 5.1 评论表 `comment`

| 字段 | 说明 |
|---|---|
| `id` | 主键 |
| `target_type` | flow_run / flow_run_node |
| `target_id` | 目标 ID |
| `author_person_id` | 评论人 |
| `content` | 评论内容 |
| `is_resolved` | 是否已解决 |
| `created_at` | 创建时间 |
| `updated_at` | 更新时间 |

### 5.2 附件表 `attachment`

| 字段 | 说明 |
|---|---|
| `id` | 主键 |
| `target_type` | flow_run / flow_run_node / comment / deliverable |
| `target_id` | 目标 ID |
| `file_name` | 文件名 |
| `file_url` | 存储地址 |
| `file_size` | 文件大小 |
| `file_type` | 文件类型 |
| `uploaded_by` | 上传人 |
| `created_at` | 创建时间 |

### 5.3 节点执行日志表 `flow_run_node_log`

| 字段 | 说明 |
|---|---|
| `id` | 主键 |
| `run_id` | 流程实例 ID |
| `run_node_id` | 实例节点 ID |
| `log_type` | info / warn / error / action |
| `operator_type` | person / agent / system |
| `operator_id` | 操作者 ID |
| `content` | 日志内容 |
| `extra_json` | 扩展信息 |
| `created_at` | 创建时间 |

---

## 6. 业务规则

* 评论不直接改变流程状态
* 附件必须绑定到明确目标对象
* 节点附件不能展示在其他节点
* 评论附件不能展示在其他评论
* 日志只能追加，不能编辑和删除
* `@人` MVP 阶段只作为文本保存
* MVP 阶段不做站内信、短信、邮件、企微、飞书通知

---

## 7. 验收标准

* 流程评论能发布并归属于流程
* 节点评论能发布并归属于节点
* 流程评论和节点评论不会混淆
* 附件能上传并绑定到正确目标对象
* 刷新页面后评论和附件仍可见
* 节点关键动作都会写入日志
* 日志能展示操作者、动作、时间和内容
* 龙虾模拟执行日志能和人工操作日志区分
