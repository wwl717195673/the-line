# 01 基础数据模块

---

## 1. 模块目标

基础数据模块负责维护人员和龙虾两个核心对象。

人员用于明确流程和节点的真实责任主体。龙虾用于表达节点上的执行主体，并为后续接入真实 OpenClaw 留出结构。

MVP 阶段不做组织架构、部门层级、多工作空间隔离和复杂权限矩阵，只保证人员和龙虾可以被流程节点引用。

---

## 2. 用户与入口

使用角色：

- 管理员
- 发起人
- 节点责任人
- 审核人

页面入口：

- 资源中心 - 人员管理
- 资源中心 - 龙虾管理
- 节点详情 - 责任人展示
- 节点详情 - 绑定龙虾展示

---

## 3. 前端页面需求

### 3.1 人员管理列表页

页面目标：管理员可以查看当前系统内可被分配到流程节点的人员。

列表字段：

- 姓名
- 邮箱
- 默认角色类型
- 状态
- 创建时间
- 更新时间
- 操作

操作项：

- 新建人员
- 编辑人员
- 停用人员

交互规则：

- 默认只展示启用人员
- 需要提供“显示停用人员”筛选
- 停用人员在列表中需要有明确状态标识
- 停用人员不能在新节点责任人选择中出现
- 历史流程中仍展示停用人员姓名

### 3.2 人员创建和编辑弹窗

页面目标：管理员可以维护人员基础信息。

字段：

- 姓名，必填
- 邮箱，必填
- 默认角色类型，必填
- 状态，必填

校验规则：

- 姓名不能为空
- 邮箱不能为空
- 邮箱格式需要合法
- 默认角色类型只能选择系统内置角色
- 新建人员默认状态为启用

### 3.3 龙虾管理列表页

页面目标：管理员可以查看当前系统中可绑定到节点的龙虾。

列表字段：

- 龙虾名称
- 龙虾编码
- 来源
- 版本
- 维护人
- 状态
- 创建时间
- 更新时间
- 操作

操作项：

- 新建龙虾
- 编辑龙虾
- 停用龙虾

交互规则：

- 默认只展示启用龙虾
- 需要提供“显示停用龙虾”筛选
- 停用龙虾不能被新节点绑定
- 已绑定到历史节点的龙虾仍保留展示

### 3.4 龙虾创建和编辑弹窗

字段：

- 龙虾名称，必填
- 龙虾编码，必填
- 来源，默认 `openclaw`
- 版本，必填
- 维护人，必填
- 配置快照，选填 JSON
- 状态，必填

校验规则：

- 龙虾名称不能为空
- 龙虾编码不能为空
- 同一个系统内龙虾编码必须唯一
- 配置快照如果填写，必须是合法 JSON

## 4. 后端功能需求

### 4.1 人员创建

接口建议：`POST /api/persons`

输入：

- `name`
- `email`
- `role_type`
- `status`

处理规则：

- 校验必填字段
- 校验邮箱格式
- 创建 `person` 记录
- 返回人员详情

### 4.2 人员更新

接口建议：`PUT /api/persons/{id}`

处理规则：

- 只允许更新姓名、邮箱、默认角色类型和状态
- 如果人员不存在，返回错误
- 如果人员已被历史流程引用，仍允许更新展示信息
- 更新后刷新 `updated_at`

### 4.3 人员停用

接口建议：`POST /api/persons/{id}/disable`

处理规则：

- 将 `person.status` 更新为停用
- 不删除人员记录
- 不回写历史流程和历史节点
- 后续责任人选择接口不返回停用人员

### 4.4 人员列表

接口建议：`GET /api/persons`

查询参数：

- `status`
- `keyword`

处理规则：

- 默认返回启用人员
- 支持按姓名或邮箱模糊查询
- 用于资源中心列表和责任人选择

### 4.5 龙虾创建

接口建议：`POST /api/agents`

输入：

- `name`
- `code`
- `provider`
- `version`
- `owner_person_id`
- `config_json`
- `status`

处理规则：

- 校验必填字段
- 校验 `code` 唯一
- 校验维护人存在且启用
- 校验 `config_json` 格式
- 创建 `agent` 记录

### 4.6 龙虾更新

接口建议：`PUT /api/agents/{id}`

处理规则：

- 只允许更新名称、版本、维护人、配置快照和状态
- `code` 原则上不建议修改
- 如果已被历史节点引用，仍允许更新展示信息

### 4.7 龙虾停用

接口建议：`POST /api/agents/{id}/disable`

处理规则：

- 将 `agent.status` 更新为停用
- 不删除龙虾记录
- 不影响历史节点展示
- 后续节点绑定下拉不返回停用龙虾

### 4.8 龙虾列表

接口建议：`GET /api/agents`

查询参数：

- `status`
- `keyword`

处理规则：

- 默认返回启用龙虾
- 支持按名称或编码查询
- 用于资源中心列表和节点绑定选择

## 5. 数据落点

### 5.1 人员表 `person`


| 字段           | 说明     |
| ------------ | ------ |
| `id`         | 主键     |
| `name`       | 姓名     |
| `email`      | 邮箱     |
| `role_type`  | 默认角色类型 |
| `status`     | 状态     |
| `created_at` | 创建时间   |
| `updated_at` | 更新时间   |


### 5.2 龙虾表 `agent`


| 字段                | 说明   |
| ----------------- | ---- |
| `id`              | 主键   |
| `name`            | 龙虾名称 |
| `code`            | 龙虾编码 |
| `provider`        | 来源   |
| `version`         | 版本   |
| `owner_person_id` | 维护人  |
| `config_json`     | 配置快照 |
| `status`          | 状态   |
| `created_at`      | 创建时间 |
| `updated_at`      | 更新时间 |


---

## 6. 验收标准

- 管理员能创建、编辑、停用人员
- 停用人员不能再作为新节点责任人被选择
- 历史流程中仍能展示停用人员姓名
- 管理员能创建、编辑、停用龙虾
- 停用龙虾不能再绑定到新节点
- 历史节点中仍能展示已绑定的停用龙虾
- 节点详情能读取并展示责任人和绑定龙虾
- 点击“运行龙虾”时能读取当前绑定龙虾信息
