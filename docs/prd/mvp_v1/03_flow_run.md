# 03 流程实例模块

---

## 1. 模块目标

流程实例模块负责把固定模板变成一次真实业务流程，并控制流程在节点之间串行推进。

MVP V1 只支持“班主任甩班申请”固定模板的串行流转，不支持并行、条件分支和复杂回退。

---

## 2. 用户与入口

使用角色：

* 发起人
* 节点责任人
* 审核人
* 管理员
* 观察者

页面入口：

* 工作台 - 发起流程
* 模板中心 - 使用模板
* 流程中心 - 全部流程
* 流程中心 - 我发起的
* 流程中心 - 待我处理
* 流程详情页

---

## 3. 前端页面需求

### 3.1 流程发起页

页面目标：用户基于固定模板提交一次甩班申请。

展示内容：

* 模板名称
* 模板说明
* 流程节点预览
* 发起表单

发起表单字段：

* 实例标题，必填
* 申请原因，必填
* 涉及班级，必填
* 当前班主任，必填
* 期望处理时间，必填
* 补充说明，选填
* 附件，选填

操作项：

* 提交发起
* 取消返回

交互规则：

* 必填项缺失时不能提交
* 提交成功后跳转流程详情页
* 提交失败时展示明确错误原因

### 3.2 流程实例列表页

列表字段：

* 实例名称
* 模板名称
* 当前节点
* 当前责任人
* 状态
* 发起人
* 最近更新时间
* 操作

筛选项：

* 状态
* 负责人
* 发起人

操作项：

* 查看流程
* 取消流程

交互规则：

* 默认按最近更新时间倒序
* “我发起的”只展示当前用户发起的流程
* “待我处理”只展示当前节点责任人为当前用户的流程
* 已完成和已取消流程不展示取消操作

### 3.3 流程详情页

展示内容：

* 流程标题
* 模板名称
* 发起人
* 当前状态
* 当前节点
* 当前责任人
* 节点时间线
* 节点详情侧栏
* 流程评论区

交互规则：

* 当前节点必须高亮
* 已完成节点展示完成状态
* 未开始节点展示未开始状态
* 点击节点后右侧切换节点详情
* 已取消流程不展示处理按钮
* 已完成流程展示生成交付物入口

### 3.4 取消流程弹窗

字段：

* 取消原因，必填

交互规则：

* 只有发起人或管理员可以取消
* 已完成流程不能取消
* 已取消流程不能重复取消
* 取消后回到流程详情页并展示已取消状态

## 4. 后端功能需求

### 4.1 发起流程实例

接口建议：`POST /api/runs`

输入：

* `template_id`
* `title`
* `input_payload_json`

处理规则：

* 校验模板存在且状态为 `published`
* 校验发起人有效
* 校验发起表单必填字段
* 创建 `flow_run`
* 查询模板节点
* 批量创建 `flow_run_node`
* 第一个节点状态为 `ready`
* 其他节点状态为 `not_started`
* 写入流程创建日志
* 返回流程实例详情

状态写入：

* `flow_run.current_status = running`
* `flow_run.current_node_code = first_node.node_code`
* `flow_run.started_at = now`

### 4.2 查询流程实例列表

接口建议：`GET /api/runs`

查询参数：

* `status`
* `owner_person_id`
* `initiator_person_id`
* `scope`

处理规则：

* `scope=all` 返回用户可见流程
* `scope=initiated_by_me` 返回当前用户发起的流程
* `scope=todo` 返回当前节点责任人为当前用户的流程
* 列表需要关联当前节点责任人和发起人姓名
* 默认按 `updated_at` 倒序

### 4.3 查询流程实例详情

接口建议：`GET /api/runs/{id}`

返回内容：

* 流程基础信息
* 当前节点信息
* 节点列表
* 节点责任人
* 节点绑定龙虾
* 流程评论
* 关键日志摘要

处理规则：

* 节点按 `sort_order` 或生成顺序返回
* 当前节点需要明确标识
* 已取消流程返回只读状态

### 4.4 取消流程

接口建议：`POST /api/runs/{id}/cancel`

输入：

* `reason`

处理规则：

* 校验当前用户是发起人或管理员
* 校验流程不是 `completed`
* 校验流程不是 `cancelled`
* 校验取消原因不能为空
* 更新流程状态为 `cancelled`
* 未完成节点不再允许处理
* 写入取消日志

### 4.5 流程串行推进

触发入口：

* 节点标记完成
* 审核通过

处理规则：

* 当前节点状态更新为 `done`
* 当前节点写入 `completed_at`
* 查询当前节点之后的下一个节点
* 如果存在下一个节点，将下一个节点状态改为 `ready`
* 如果存在下一个节点，更新流程 `current_node_code`
* 如果不存在下一个节点，更新流程状态为 `completed`
* 如果流程完成，写入 `completed_at`
* 写入节点推进日志

## 6. 业务规则

* MVP 只支持串行推进
* 当前节点完成后才能激活下一个节点
* 已取消流程不能处理节点
* 已完成流程不能取消
* 取消流程必须填写原因
* 流程完成后才能生成交付物
* 模板变更不影响已创建的流程实例节点
* 节点进入 `waiting_confirm`、`waiting_material` 或 `rejected` 时，流程状态为 `waiting`
* 节点标记异常时，流程状态为 `blocked`
* 节点完成并激活下一个节点时，流程状态为 `running`
* 最后一个节点完成时，流程状态为 `completed`

---

## 7. 验收标准

* 用户能基于“班主任甩班申请”发起流程
* 发起后自动生成 9 个实例节点
* 第一个节点为 `ready`，其他节点为 `not_started`
* 流程列表能展示当前节点和当前责任人
* “我发起的”和“待我处理”筛选结果正确
* 流程详情能展示完整节点时间线
* 完成当前节点后下一个节点自动变为 `ready`
* 完成最后一个节点后流程状态变为 `completed`
* 发起人或管理员能取消未完成流程
