# 接入虾线平台

我需要你帮我接入虾线（the-line）工作流平台。虾线是一个流程编排系统，它会给你派发任务，你负责执行并回传结果。

整个过程只需要 HTTP 调用，不需要安装任何东西。

## 接入信息

- 平台地址: `http://localhost:8080`
- 注册码: `TL-52FB-DCA8`（24小时内有效）

---

## 第一步：注册

向平台注册，拿到你的身份标识。请生成一个随机字符串作为你的 instance_fingerprint。

```http
POST http://localhost:8080/api/integrations/openclaw/register
Content-Type: application/json

{
  "protocol_version": 1,
  "registration_code": "TL-52FB-DCA8",
  "bridge_version": "0.1.0",
  "instance_fingerprint": "你的随机唯一标识",
  "display_name": "你给自己取个名字",
  "bound_agent_id": "default-agent",
  "callback_url": "",
  "capabilities": {"draft_generation": true, "agent_execute": true}
}
```

成功后你会拿到 `integration_id`（一个数字），后续所有操作都需要它。

---

## 第二步：进入工作循环

注册成功后，按照下面的循环持续工作：

### 2.1 拉取待执行任务

```http
GET http://localhost:8080/api/integrations/openclaw/{integration_id}/pending-tasks
```

- 返回数组。如果是空数组 `[]`，说明暂时没有任务，等几秒再查。
- 如果有任务，每个任务包含 `id`、`task_type`、`input_json` 等字段。

### 2.2 认领任务

拿到任务后，先认领（防止重复执行）：

```http
POST http://localhost:8080/api/integrations/openclaw/{integration_id}/claim-task/{task_id}
```

认领成功后任务状态变为 `running`。

### 2.3 执行任务

根据任务信息执行工作：

| task_type | 你要做什么 |
|-----------|-----------|
| `query` | 查询数据，整理结果 |
| `batch_operation` | 批量执行操作，统计成功/失败数 |
| `export` | 导出数据或生成文件 |

具体要做什么看 `input_json` 里的内容。

### 2.4 回传结果

执行完毕后，把结果告诉平台：

```http
POST http://localhost:8080/api/agent-tasks/{task_id}/receipt
Content-Type: application/json

{
  "agent_id": 任务里的agent_id,
  "status": "completed",
  "started_at": "开始执行的时间(ISO8601格式)",
  "finished_at": "完成的时间(ISO8601格式)",
  "summary": "用一句话描述你做了什么",
  "result": { 结构化的执行结果 },
  "artifacts": [],
  "logs": ["执行步骤1", "执行步骤2", "..."],
  "error_message": ""
}
```

**status 可选值：**

| 值 | 什么时候用 |
|---|----------|
| `completed` | 执行成功完成 |
| `needs_review` | 做完了但需要人来审核 |
| `blocked` | 发现风险，不敢继续，需要人介入 |
| `failed` | 执行失败了 |

**重要原则：如果你在执行中发现有风险（数据不对、权限不够、可能造成不可逆影响），不要硬做。回传 `blocked`，说明原因，让人来决定。**

### 2.5 心跳

每 60 秒告诉平台你还在线：

```http
POST http://localhost:8080/api/integrations/openclaw/heartbeat
Content-Type: application/json

{
  "integration_id": 你的integration_id,
  "bridge_version": "0.1.0",
  "status": "healthy",
  "active_runs_count": 0
}
```

## 整体流程总结

```
注册 → 拿到 integration_id
       ↓
  ┌──→ 拉取任务
  │    有任务？─── 没有 → 等 5 秒 → 回到拉取
  │    ↓ 有
  │    认领任务
  │    执行任务
  │    回传结果
  │    ↓
  │    发送心跳（每60秒一次）
  └────┘
```

## 验证接入是否成功

注册后查一下自己的状态：

```http
GET http://localhost:8080/api/integrations/openclaw/{integration_id}
```

应该能看到 `status` 为 `active`。

## 常见错误

| 响应 | 原因 | 怎么办 |
|------|------|--------|
| 400 注册码无效 | 码错了或过期了 | 让我重新生成一个 |
| 400 绑定的龙虾编码不存在 | 平台上没有这个 agent | 把 bound_agent_id 改成平台已有的 |
| 404 集成实例不存在 | integration_id 不对 | 检查注册时拿到的 ID |
| 409 任务状态不是待执行 | 任务已经被认领了 | 跳过，拉下一个 |
